# 3. その他仕様（要素に収まらない共通ルールや制約）

## 3-1. 認証・セキュリティ仕様
- **認証方式**: メールアドレス + パスワード認証
- **対応アカウント**: 
  - 一般管理者: admin@ansteype.com / admin123
  - Festal管理者: festal.admin@ansteype.com / festal123  
  - テスト用: test@ansteype.com / test
- **ログイン状態保持**: LocalStorageに以下を保存
  - admin_logged_in: ログイン状態
  - admin_id: 管理者ID
  - admin_type: 管理者タイプ（general/festal/test）
  - admin_email: メールアドレス
  - admin_name: 管理者名
  - admin_remember: 保持フラグ（チェック時のみ）

## 3-2. バリデーション仕様
- **メールアドレス**: 必須、メール形式（正規表現: /^[^\s@]+@[^\s@]+\.[^\s@]+$/）
- **パスワード**: 必須
- **エラーメッセージ**:
  - 未入力: 「メールアドレスとパスワードを入力してください」
  - メール形式エラー: 「正しいメールアドレス形式で入力してください」
  - 認証エラー: 「メールアドレスまたはパスワードが正しくありません」

## 3-3. UI/UX仕様
- **レスポンシブ**: Container maxWidth="sm"（最大幅600px）で中央配置
- **カード設計**: 最大幅400px、影付き、角丸2px
- **ローディング状態**: ボタンテキスト変更（「ログイン中...」）、フィールド無効化
- **キーボード操作**: Enterキーでログイン実行可能
- **アクセシビリティ**: 全操作キーボード対応、適切なaria-label設定

## 3-4. パスワードリセット仕様
- **対象メール**: 登録済み管理者メールアドレスのみ
- **送信処理**: モック実装（1.5秒遅延）
- **成功時**: 成功メッセージ表示、詳細な手順説明
- **エラー時**: 「登録されていないメールアドレスです」表示
- **ダイアログ**: 最大幅sm、フルワイド、状態管理完備

## 3-5. 画面遷移仕様
- **ログイン成功**: /tier-2dealer/admin/shifts へ自動遷移
- **スタッフログイン**: /tier-2dealer/staff/login へ遷移
- **認証状態確認**: ページロード時にLocalStorageチェック

## 3-6. デモ・開発仕様
- **認証情報表示**: グレー背景で3つのデモアカウント情報を表示
- **モック認証**: 1秒遅延でリアルな認証体験を提供
- **ログ出力**: パスワードリセット時にコンソールログ出力
