# 3. その他仕様（要素に収まらない共通ルールや制約）

- **認証管理**: スタッフログイン必須、セッション管理
- **データ生成**: ダミーデータによる確定シフト情報の動的生成
- **ステータス管理**: 
  - 希望ステータス: ○（緑）、×（赤）、-（グレー）
  - 確定ステータス: 確定（緑）、現場未確定（黄）、詳細未確定（オレンジ）、休み（グレー）
- **レスポンシブ対応**: 
  - デスクトップ: フルテーブル表示
  - タブレット: 横スクロール対応
  - スマホ: カード形式表示
- **アクセシビリティ**: 
  - WCAG 2.2 AA 準拠
  - キーボード操作完全対応
  - コントラスト4.5:1以上
  - スクリーンリーダー対応
- **UI/UX**: 
  - 稼働詳細ドロワーは右スライドイン形式、幅900px
  - テーブル行のホバー効果
  - ローディング状態表示
  - エラー状態のフォールバック表示
- **データ表示**: 
  - 稼働時間: 平日11:00-19:00、土日10:00-18:00
  - 日当: 平日¥15,000、土日¥18,000
  - 集合時間: 稼働開始30分前
  - 代理店: ANSTEYPE、イベントプロ、マーケティングラボ等
  - 稼働場所: 各エリアの代表的な場所を表示
- **決定論的データ生成**: 
  - スタッフIDと日付に基づく疑似ランダム生成
  - 希望○を22日分、残りを×で配分
  - 希望に基づく確定結果の決定ロジック
- **コミュニケーション機能**: 
  - 管理者とのリアルタイムメッセージ
  - 既読・未読状態管理
  - ファイル添付未対応
- **パフォーマンス**: 
  - 効率的なダミーデータ生成
  - 最小限のAPI通信
  - メモ化による再レンダリング最適化
- **セキュリティ**: 
  - スタッフ認証必須
  - 他スタッフ情報の非表示
  - セッションタイムアウト対応
- **注意事項**: 
  - 現在はダミーデータによる表示
  - 実際のAPI連携は未実装
  - リアルタイム更新機能は未実装
  - 土日は背景色をオレンジ系に変更
